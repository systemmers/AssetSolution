# Part 2: 하드코딩 제거 리팩토링 상세 계획

## 📋 프로젝트 개요

### 🎯 목표
- Python 및 JavaScript 전반에 걸친 하드코딩 값들을 constants.py와 JavaScript 설정으로 통합
- 비즈니스 로직에서 하드코딩된 값들을 repositories 또는 설정 파일로 이관
- 중복된 설정값들을 일관성 있게 관리
- 유지보수성과 확장성 향상

### 📊 현재 상황
| 분류 | 발견된 하드코딩 | 목표 | 우선순위 |
|------|---------------|------|----------|
| 타임아웃/지연 | 30000ms, 3000ms, 500ms 등 | 통합 설정 | 🔴 높음 |
| 페이지 설정 | 10, 20, 25, 50 등 | constants.py 통합 | 🟠 중간 |
| 비즈니스 값 | 대여한도 5개, 상태 ID 등 | repositories 이관 | 🔴 높음 |
| 파일 크기 | 16MB 등 | 설정 파일 통합 | 🟡 낮음 |

## 🏗️ 하드코딩 분류 및 처리 전략

### Python 하드코딩 처리
```
하드코딩 값들:
├── 비즈니스 로직 값
│   ├── 대여 한도: 5개 → repositories/business_rules.py
│   ├── 상태 ID: 3 (대여중) → constants.py 활용
│   └── 날짜 기본값: 30일 → constants.py 활용
│
├── 설정 값들
│   ├── 페이지 사이즈: 10 → constants.py 활용
│   ├── 파일 크기: 16MB → constants.py 활용
│   └── 경고 일수: 30일 → constants.py 활용
│
└── 메시지 텍스트
    ├── 에러 메시지들 → constants.py 또는 i18n
    └── 성공 메시지들 → constants.py 또는 i18n
```

### JavaScript 하드코딩 처리
```
하드코딩 값들:
├── 타임아웃/지연
│   ├── API 타임아웃: 30000ms → config.js
│   ├── 알림 지연: 3000ms → config.js
│   └── 검색 디바운스: 500ms → config.js
│
├── UI 설정
│   ├── 페이지 사이즈: 10, 20, 25, 50 → config.js
│   ├── 애니메이션: 1000ms → config.js
│   └── 차트 업데이트: 5000ms → config.js
│
└── 비즈니스 값
    ├── 자동 새로고침: 30초, 5분 → config.js
    └── 업로드 제한 → config.js
```
